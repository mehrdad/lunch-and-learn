Data and DOM Manipulations Library

@alexsergeyev, February 2012

Data Driven Documents. YES!

  • fast, small (minimalisic core and extra modules)
  • allows to template DOM element creation
  • utilizes common idea "Find something in DOM and keep it in array"
  • plays nicely with HTML5 and other tools (would also switch to Sizzle, if available)
  • very well documented (mbostock.github.com/d3/api/)

          Comparing to jQuery, D3.js is...
        

        
    
        
  • full of handy array and math utils, not just DOM operations
        
  • half the size 
        
  • ignorant of pre-IE9 and other old browsers
            
    (having Sizzle saves you some pain but SVG is not
                 supported in those anyway)
        

        

        Read apples vs oranges to find more things to compare.
        

        Both are free! Modified BSD License (3-clause, BSD-new) for D3.js and MIT and GPL for jQuery.
        

      


      

        

          Modules in D3.js
        

        
Core, d3.js

        
    
          
  • Selections, transitions, array functions
          
  • Loaders (xml, xhr, txt, xml, json, csv)
          
  • SVG shapes, axes and events
          
  • Scales (linear, power, log, etc), enumerations, colorings
          
  • Math, Color, String and XML utils
        

        
Time, d3.time.js

        
Various date/time calculation and formatting utils. Time scales and intervals.

        
Layout, d3.layout.js

        
Various classes to convert data to meaningful numbers for graps. Stack, tree, pie, and others.

        
Geography, d3.geo.js

        
GeoJSON parsers and shape drawing utlis. Map projections and specialized scales

        
Others Are Coming Up!

        
Geom: Specific layouts for exotic charts;

        Chart: Box, bullet, horizon charts (so far)

          Should I stop using my pretty-pretty graph tool?
        

        

          No, just consider following about those tools:
        

        
    
          
  • They render and put standard UI around your charts
          
    D3 gives you data manipulation tools
    
          
  • Some of them can do just so much and will behave irradically if you need 100
          charts on same page or 10000 datapoints in one chart
          
    It's harder to put "too much" to DOM or stress CSS3 engines
    
          
  • I wonder how many of them are extensivly tested and expandable
          
    Similar to jQuery D3.js uses node.js to compile/test/build code
    
        

      


      

        

          Too much good things about D3.js? Here are negatives:
        

        
    
        
  • Many browsers are not supported
          
    
          Some people render SVGs into bitmaps using AJAX and other tools.
          
    
        
  • Extension base and third party solutions just started to appear
          
    
          It's like with jQuery, think twice before you use plugin or
          library based on D3.js
          
    
        
  • There is a huge learning curve to understand all of the D3 potential
          
    
          It's not a magic wand but there is usually a way to do more than you think is possible.
    
          
    
        

      


      

        

          How to start?
        

        
Clone git repository or download a release, explore examples!
        
(it's better to have a web-server to look at them)
        
Modify them with your data or try to visualize differently.
        
(feel free to improve them and offer your ideas back to developers)
        
Don't forget to use in-browser tools to peek behind the curtains.
        
Join D3 discussion group to find
        inspiration or to solve a problem.
