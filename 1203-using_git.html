Introduction

Git is revision control system. You use one to make sure changes you do over time could be restored.

Git positions itself as distributed, decentralized RCS.

It stores all revision information in BLOBs optimized for fast operations. Every working copy is effectively a fully functional fork of git DAG structures and work files extracted from them.

...but directed graphs are so cool!

  • All branches and tags are just node labels.
  • "Revision" is the product of previous state and applied diff.

It makes merging process smarter and easier to execute! Git can also merge more than pair of branches at the same time.

Let's do it! Creating Git Repository (step 1) 

$ mkdir myrepo; cd myrepo
$ git init .
Initialized empty Git repository in ./.git/
$ echo "Hello World" > test.txt
$ git add test.txt
$ git commit test.txt -m 'Adding a greeting'
[master (root-commit) 4be30d5] Adding a greeting
 1 files changed, 1 insertions(+), 0 deletions(-)
 create mode 100644 test.txt

Try it with "git status" after every line.

Try to repeat some git commands after you've already done them.

Let's do it! Branching and merging (step 2) 

$ git checkout -b patch
Switched to a new branch 'patch'
$ git branch -v
  master 0ac2b99 Adding a greeting
* patch  0ac2b99 Adding a greeting
$ echo 'Git is cool' >> test.txt
$ git commit -am 'Extending the greeting'
[patch ab58480] Extending the greeting
 1 files changed, 1 insertions(+), 0 deletions(-)
$ git checkout master
Switched to branch 'master'
$ git branch -v
* master 0ac2b99 Adding a greeting
  patch  ab58480 Extending the greeting
$ git merge patch
Updating 0ac2b99..ab58480
Fast-forward
 test.txt |    1 +
 1 files changed, 1 insertions(+), 0 deletions(-)
$ git branch -v
* master ab58480 Extending the greeting
  patch  ab58480 Extending the greeting
$ git branch -D patch
Deleted branch patch (was ab58480).

"Fast-forward" is a simple merge. Git just updated the label-node reference!
